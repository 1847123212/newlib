The OpenRISC architecture recently moved towards multicore implementations. There is a toolchain that can execute code on multiple cores in a symmetric fashion.

If you build the toolchain manually, you should use a different prefix, such as /opt/toolchains/or1k-elf-multicore and you need to build newlib with the environment variable CFLAGS="-D__OR1K_MULTICORE__":

../newlib/configure --target=or1k-elf --prefix=$PREFIX CFLAGS="-D__OR1K_MULTICORE__"

Releases

You can install pre-compiled toolchains and install them on your Linux system. We have prebuilt-toolchains for releases of the different components that need to be installed to /opt/or1k-elf. You can find all releases here. The latest release:

Development builds

There are continuous builds from the current upstream versions of binutils, gdb and newlib along with the OpenRISC gcc port. They are automatically executed after source code changes and their results are automatically uploaded here.

For each build, the txt file describes the git revisions used in that build and the successfully built, but untested, toolchains are available for different operating systems.
